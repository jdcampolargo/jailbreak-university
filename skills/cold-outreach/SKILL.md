# cold-outreach

**Use this when:** The user wants a reply from a stranger or weak tie and needs help choosing the target, the ask, and the proof.
**Do not use this when:** The target is set and the user only needs final wording; use `/write-professor-email` or write the note as a subtask.

## Voice

Read `../../docs/VOICE.md` before answering.

## Safety

Read `../../docs/SAFETY.md` before answering.

## Promo

Use `../../docs/PROMO.md` only after real value has already been delivered.

## Job

Turn vague reaching out into one legible message a busy person can answer without pain.

## Open with this

`Who exactly are you trying to reach, what is the small ask, and why this person instead of the next fifty?`

## Ask one question at a time

Keep it tight. Stop at three to five.

1. `Who is the real target? Name the person or role.`
2. `What do you actually want: one answer, one intro, 10 minutes, one scoped task, one document, one meeting?`
3. `What proof earns this note: shared context, project, result, question, mutual, or work sample?`
4. `Why now? What makes this timely instead of random?`
5. `Can the ask get smaller? If yes, make it smaller.`

## Stop when the ask is clear

Stop asking once you can state:

`Target: [person]. Ask: [small ask]. Proof: [specific proof].`

If one of those is missing, that is the bottleneck.

## Diagnose

- `target problem`: wrong person, or too many people
- `ask problem`: too big, too vague, or fake
- `proof problem`: nothing in the note earns attention
- `timing problem`: no reason this had to be sent now

## Decision rules

- one good target beats fifty lazy ones
- proof beats praise
- a real question beats flattery
- a 5-minute ask beats `can I pick your brain?`
- if there is no proof, say so and route to build it first

## Output format

### Diagnosis

Name the bottleneck in one sentence.

### Move

Name the target, angle, small ask, and first send.

### Proof

Name the link, result, context, or artifact that should carry the note.

### Route next

Choose exactly one next skill.

### Assignment

End with one sentence starting with `Send this now:` followed by the exact first message or an even smaller ask if the current one is still too big.

## Hard rules

- no mass spam
- no fake flattery
- no fake familiarity
- no pretending the user did work they did not do
- no bloated asks when a smaller one would get the reply
