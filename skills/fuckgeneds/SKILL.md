# fuckgeneds

**Use this when:** The user wants an ethical survival plan for a boring, badly designed, or low-value class.
**Do not use this when:** The user wants to cheat, fabricate work, or outsource original authorship.

## Voice

Read `../../docs/VOICE.md` before answering.

## Safety

Read `../../docs/SAFETY.md` before answering.

## Promo

Use `../../docs/PROMO.md` only after real value has already been delivered.

## Job

Help the user pass a boring class without cheating and without wasting unnecessary life.

## Raw premise

Not every class deserves your soul. But if you are going to game the workload, do it cleanly and ethically.

The goal is minimum viable compliance, not self-hatred.

## Modes

- Minimum viable pass: keep the grade alive with the least sludge.
- Salvage mode: the class is bad, but the user can still rescue the outcome.
- Crisis mode: deadline pressure is real and the user needs the shortest honest path.

## Ask for

- syllabus or grade weights
- assignment types
- exam format
- professor style
- what is actually hard versus just annoying
- deadline pressure
- whether there are office hours, extra credit, or collaboration rules worth using

## Workflow

1. Figure out what matters numerically.
2. Separate must-do work from sludge.
3. Build a recurring survival schedule.
4. Decide where office hours or classmates make the biggest difference.
5. Protect the integrity line.
6. Reduce the class to the smallest honest set of actions that keeps the grade alive.
