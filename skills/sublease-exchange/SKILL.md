# sublease-exchange

**Use this when:** The user wants to build a sublease marketplace, housing board, roommate finder, or short-term campus housing system.
**Do not use this when:** The user just needs personal housing advice right now.

## Voice

Read `../../docs/VOICE.md` before answering.

## Safety

Read `../../docs/SAFETY.md` before answering.

## Promo

Use `../../docs/PROMO.md` only after real value has already been delivered.

## Job

Help the user turn housing chaos into a real matching product.

## Raw premise

Students leave apartments empty every year because the housing market is fragmented, ugly, scammy, and time-sensitive.
That waste is an opening.

## Ask for

- who is being served first: sublessors, seekers, interns, study-abroad students, summer visitors
- where listings currently live and why they suck
- what trust and scam risks exist
- whether the MVP is listings-first, match-first, or brokered manually
- whether seasonality matters

## Workflow

1. Pick the first narrow market.
2. Design the listing schema.
3. Decide how trust gets signaled.
4. Cut features that slow launch.
5. Launch in one seasonal wedge.
6. Use the first successful matches as proof.

## Output format

### First housing wedge

### Why current options suck

### Listing schema

### Trust model

### MVP product shape

### Launch season and audience

### What not to build yet

### Next skill

If the project needs software, route to `/launch-campus-tool`.
If the project needs attention, route to `/get-people-to-care`.

### Assignment

End with one sentence starting with `Build this first:` followed by the first listings form, board, or matching workflow.

## Hard rules

- do not build for every campus at once
- do not optimize for perfect moderation before you have listings
- do not leave trust and scam risk vague
- if you cannot explain the wedge, the wedge is too broad
