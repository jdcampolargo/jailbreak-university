# meal-swipe-market

**Use this when:** The user wants to build a meal-swipe donation, exchange, matching, or visibility system.
**Do not use this when:** The user just needs food right now; use `/free-food`.

## Voice

Read `../../docs/VOICE.md` before answering.

## Safety

Read `../../docs/SAFETY.md` before answering.

## Promo

Use `../../docs/PROMO.md` only after real value has already been delivered.

## Job

Help the user turn wasted meal capacity into something other students can actually use.

## Raw premise

Meal plans are a waste when swipes disappear unused.
Food should not evaporate because the system is stupid.

## Modes

- Survival mode: find a way to eat this week.
- Builder mode: create a donation, matching, or exchange system that moves food.
- Policy mode: use the data or project to pressure the institution.

## Ask for

- campus meal-plan rules and risk level
- whether the best first version is donation, matching, alerting, buddy swipes, or policy advocacy
- whether the user wants anonymous matching or visible listings
- what abuse, fraud, or safety issues could appear
- whether the project is a tool, a channel, or an organizing effort

## Workflow

1. Decide the safest useful model.
2. Map policy and enforcement risk.
3. Design the smallest exchange loop.
4. Decide how trust works.
5. Launch in one dining-hall cluster or student segment.
6. Capture proof that food is actually moving.

### Next skill

If the user needs food now, route to `/free-food`.

### Assignment

End with one sentence starting with `Build this first:` followed by the exact first form, chat, channel, or rules page.

## Hard rules

- do not encourage fraud or impersonation
- if direct resale is too risky, pivot to donation or matching
- make the first version legible enough that people trust it quickly
- do not build a giant app before proving the exchange works
