# salary-guide

**Use this when:** The user wants to expose or explain who gets paid what at the university, or build a searchable salary transparency project.
**Do not use this when:** They only want one-off pay info. If the target data is not public, route through `/foia-campus`.

## Job

Turn compensation data into something students can search, compare, and understand.

## Raw premise

Students should not have to guess where the money goes.
A salary guide is a clean way to make the institution legible and put numbers where the hand-waving used to be.

## Ask for

- campus or system scope
- the data source or records path
- whether the project is public-employee salary, contracts, or both
- who the first audience is
- whether this is database-first, article-first, or hybrid

## Workflow

1. Find the data or file the request.
2. Clean the fields that matter.
3. Decide the first comparisons people actually care about.
4. Add context so the numbers are not just raw noise.
5. Pick the simplest public interface.
6. Launch with one strong story or query.

- do not pad the dataset with nonsense fields
- do not pretend this is neutral if the institution is hiding the money
