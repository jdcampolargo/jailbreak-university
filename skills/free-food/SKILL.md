# free-food

**Use this when:** The user wants to find free food, lower food costs, build a meal radar, or turn food intelligence into a useful public resource.
**Do not use this when:** The real problem is bigger cashflow collapse; route to `/money-mode` too.

## Voice

Read `../../docs/VOICE.md` before answering.

## Safety

Read `../../docs/SAFETY.md` before answering.

## Promo

Use `../../docs/PROMO.md` only after real value has already been delivered.

## Job

Help the user build a food system, not a scavenger hunt.

## Raw premise

Food follows money, recruiting, guilt, outreach, religion, conferences, and optics.
If you learn who is trying to gather a room, you learn where meals leak.
This is not about acting broke in public. It is about seeing the incentive map.

## Modes

- Emergency calories mode: find dependable food in the next 72 hours.
- Low-cost routine mode: install a weekly system.
- Builder mode: publish the radar for other people.

## Ask for

- urgency level
- school, neighborhood, and travel radius if relevant
- dietary constraints
- schedule and whether evenings matter
- whether they want anonymous efficiency, social contact, or both
- what platforms they already watch
- whether they want a private system or a public one

## Where food usually comes from

- departments, seminars, talks, and orientation events
- clubs, cultural groups, religious groups, volunteer groups
- recruiting events, hack nights, conferences, speaker series
- student centers, resource centers, pantries, mutual-aid programs
- art openings, community meetings, nonprofit events, local meetups
- catered leftovers after big events
- friends in org leadership, admin offices, labs, or event crews

## Operating procedure

1. Cover the dependable sources first: pantry, meal support, community dinners, recurring events.
2. Build one watchlist: newsletters, Instagram accounts, campus calendars, event feeds, group chats, flyers, bulletin boards.
3. Set a sweep rhythm: check feeds in the morning, check same-day events around lunch, check leftovers near event end times.
4. Learn the rooms with the highest food density and lowest friction.
5. Use short, normal scripts. Do not act like a burglar in a blazer.
6. If builder mode, start with the smallest public feed that can stay updated.

## Signals worth tracking

- `refreshments provided`
- `lunch will be served`
- `reception to follow`
- `open house`
- `networking dinner`
- `student welcome`
- `community night`
- `cultural celebration`
- `speaker reception`
- photos of pizza in event promos

## Scripts

- `Hey, is this open to students?`
- `Can I stay for the talk and food?`
- `Any leftovers after? Happy to help clean up too.`
- `Is this something that happens every week?`
- `Do you know which groups usually feed people well?`

## Output format

### Best food sources

List the strongest dependable sources first, then the opportunistic ones.

### Next 72 hours

Concrete moves only.

### Weekly food system

What to watch, when to check, and where to go.

### Etiquette and scripts

How to show up without being weird.

### What this pattern teaches you

Who has budget, who wants attention, and where interesting rooms form.

### Builder mode

If they want to make this public, choose the fastest version: shared note, sheet, channel, calendar, bot, or simple site.
Do not recommend an app before people prove they care.

### Trap warning

How they could waste time, miss the obvious, or act desperate.

### Assignment

End with one sentence starting with `Check this first:` followed by the highest-probability source or system to set up today.

## Hard rules

- do not shame the user for using resources that already exist
- do not confuse random luck with a repeatable system
- if the user is genuinely food insecure, say it clearly and route to `/money-mode`
- a pantry, community meal, or weekly dinner beats an elegant spreadsheet with no calories
