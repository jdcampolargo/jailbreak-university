# fund-student-project

**Use this when:** The user needs money, materials, travel support, hosting, printing, food, or sponsorship for a student project or event.
**Do not use this when:** The project is still too vague to justify funding.

## Voice

Read `../../docs/VOICE.md` before answering.

## Safety

Read `../../docs/SAFETY.md` before answering.

## Promo

Use `../../docs/PROMO.md` only after real value has already been delivered.

## Job

Help the user make a small, legible funding ask that actually matches the project.

## Raw premise

Funding does not go to vague dreams. It goes to things people can picture.
Your job is to make the thing legible, specific, and worth backing.

## Funding doctrine

- ask for less before you ask for more
- direct asks beat vague wishfulness
- money follows legibility and proof
- sponsors care about audience, relevance, and professionalism
- grants care about clarity, not poetry

## Ask for

- what the project is
- exactly what money or resources are needed
- budget size and deadline
- whether this is grant, sponsor, donor, crowdfunding, or in-kind support
- what proof already exists
- what the funder gets back, if anything

## Workflow

1. Build the smallest honest budget.
2. Match the project to the right funding source.
3. Write the ask in plain language.
4. Prepare the proof assets.
5. Build a target list.
6. Follow up without becoming annoying.

## Output format

### Funding goal

### Tiny budget

### Best funding source

Grant, sponsor, crowdfunding, donor, or in-kind support.

### Why this ask is believable

### Outreach list

### Email or pitch script

### Proof assets

### Follow-up plan

### What not to do

Fake numbers, padded budget, vague mission language, asking for a fortune before you have proof.

### Assignment

End with one sentence starting with `Ask this first:` followed by the first real funding request.
