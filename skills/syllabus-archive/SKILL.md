# syllabus-archive

**Use this when:** The user wants a public syllabus archive, course comparison tool, professor-intel index, or class-expectations database.
**Do not use this when:** The user only needs help choosing classes right now; use `/what-to-study` or `/pick-professor`.

## Voice

Read `../../docs/VOICE.md` before answering.

## Safety

Read `../../docs/SAFETY.md` before answering.

## Promo

Use `../../docs/PROMO.md` only after real value has already been delivered.

## Job

Help the user turn scattered syllabi into a usable archive that changes how students register and prepare.

## Raw premise

Class material should not disappear into Canvas and departmental fog.
Students should be able to see what a course actually is before they are trapped in it.

## Ask for

- campus scope
- acquisition path: public pages, student submissions, department sites, or FOIA
- whether this is archive-first or comparison-first
- whether the first use case is workload, readings, grading style, or professor behavior
- what metadata matters most

## Workflow

1. Decide what counts as a syllabus for v1.
2. Choose the first acquisition path.
3. Define the metadata schema.
4. Build the smallest searchable interface.
5. Decide how new syllabi get added.
6. Launch around registration season or another painful course-selection moment.

## Output format

### Why students would care

### Best acquisition plan

### Metadata schema

### MVP interface

### Submission or FOIA path

### Professor-relations and takedown policy

### Launch angle

### Next skill

If records are missing, route to `/foia-campus`.
If the archive needs software, route to `/launch-campus-tool`.
If people need to hear about it, route to `/get-people-to-care`.

### Assignment

End with one sentence starting with `Build this first:` followed by the first 25 syllabi plan, schema, or searchable page.

## Hard rules

- start narrow by department or course cluster if needed
- do not wait for perfect completeness
- do not build a messy archive that nobody trusts
- respect takedown and copyright realities instead of being stupid about them
