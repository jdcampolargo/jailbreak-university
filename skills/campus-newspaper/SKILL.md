# campus-newspaper

**Use this when:** The user wants to start or relaunch a campus paper, newsletter, media account, reporting project, or student publication with an actual beat.
**Do not use this when:** The user only wants an anonymous advice/confessions feed; use `/anonymous-advice`. If the project is records-first, use `/foia-campus`.

## Job

Help the user build campus media that students would actually read, forward, and argue about.

## Raw premise

Most student media dies because it writes for officials, not for students.
Write about the stuff people text each other about: money, housing, food, professor energy, student projects, campus weirdness, and the parts of admin life that are not supposed to be visible.

## What this can be

- a weekly newsletter
- a student paper
- a short-form account
- an anonymous tipline with editorial judgment
- a campus beat newsletter
- a reporting project with public artifacts

## Ask for

- the beat or thesis in one sentence
- who the first audience is
- whether they want reporting, commentary, curation, or all three
- whether the byline should be public, pseudonymous, or mixed
- how often they can publish without turning into sludge
- what campus media already gets wrong

## Workflow

1. Pick one sharp editorial thesis.
2. Choose one format and one surface.
3. Plan the first ten pieces.
4. Decide how tips, submissions, and sourcing work.
5. Build distribution into the concept from day one.
6. Kill anything that sounds official, vague, or decorative.

## Hard rules

- do not try to cover everything
- do not sound official
- do not wait for institutional permission
- do not publish filler just to maintain cadence
- if the first ten pieces are weak, the project is weak
