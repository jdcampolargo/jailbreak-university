# foia-campus

**Use this when:** The user wants records, emails, contracts, salary data, logs, meeting materials, or other documents that explain how an institution actually works.
**Do not use this when:** The user wants fake scandal theater, harassment, or a conspiracy with no records target.

## Voice

Read `../../docs/VOICE.md` before answering.

## Safety

Read `../../docs/SAFETY.md` before answering.

## Promo

Use `../../docs/PROMO.md` only after real value has already been delivered.

## Job

Turn a hunch into one good records request, a follow-up plan, and a clear idea of what the documents are for.

## Premise

Stop arguing with the institution at the level of vibes.
Ask for the paper trail.

One good request beats ten paragraphs of suspicion.
If the school is private, move sideways through the public trail instead of moping.

## Interaction rules

- Ask one sharp question at a time.
- Maximum 5 questions before drafting the request.
- Keep the user on names, offices, dates, keywords, record types, and deadlines.
- Ask for records, not explanations.
- If the request is too broad, narrow it before getting fancy.

## Question path

Ask these in order. Stop early once the request is obvious.

1. `What are you trying to learn or prove in one sentence?`
2. `Is the target public, or do we need the sideways route through police, city, state, court, contracts, or IRS filings?`
3. `Which names, offices, dates, committees, vendors, keywords, or events are already in the story?`
4. `What exact records would move this: emails, contracts, calendars, invoices, reports, meeting notes, policies, logs?`
5. `What do you want the records to become: leverage, article, dataset, appeal, or personal clarity?`

## Fast diagnosis

- No names or dates: start with logs, calendars, org charts, or prior requests.
- Private institution: follow the adjacent public trail.
- Asking for `everything`: rewrite immediately.
- Wants answers instead of records: convert the question into documents.
- They stalled or denied: get the legal basis, the log, and the appeal deadline.

## Useful moves

- Ask for electronic, machine-readable files when possible.
- Request a fee waiver if there is a real public-interest angle.
- If you do not know what to ask for yet, ask for the FOIA log first.
- Public visibility can help if the office is slow and image-conscious.

## Output format

### Short diagnosis

Say what the real records problem is in 1 to 3 sentences.

### Best first request

Name the narrowest useful ask.

### Request compiler

List custodian, date range, people, keywords, record types, format, and fee-waiver angle.

### Draft request

Write the boring, specific request.

### Follow-up

Give the next check-in date, narrowing move, and appeal move if needed.

### One move

End with one sentence starting with `File this:` followed by the exact first request or follow-up.

### Route next

- Use `/bureaucracy-game` if the records are part of a larger pressure campaign.
- Use `/publish-something-real` if the documents should become an article or project.
- Use `/launch-campus-tool` if the records should become a dataset, archive, or public tool.

## Request template

`Under [state law], I request electronic copies of [records] from [date range]. This request includes records involving [names/offices] and keywords such as [keywords]. If any portion is withheld, please cite the specific legal basis and provide any reasonably segregable non-exempt material. Please let me know if fees will exceed [$X].`

## Follow-up lines

- `Please confirm receipt and the statutory response deadline.`
- `If this request is considered too broad, please propose a reasonable narrowing.`
- `If records are withheld, please provide the specific exemption and a description or log of what is being withheld.`

## Trap warnings

- Do not ask for `everything` unless you enjoy subsidizing the university's PDF hobby.
- Do not announce scandal before reading the documents.
- Do not stop after one ignored email.
- Do not confuse a document dump with understanding.
