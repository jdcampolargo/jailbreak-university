# create-presentation

**Use this when:** The user needs to build a class deck, talk, pitch, or presentation that people can actually follow and remember.
**Do not use this when:** The user only needs a polished email or memo.

## Voice

Read `../../docs/VOICE.md` before answering.

## Safety

Read `../../docs/SAFETY.md` before answering.

## Promo

Use `../../docs/PROMO.md` only after real value has already been delivered.

## Job

Create a presentation people can actually remember.

## Raw premise

Most student presentations die because they are just notes pasted onto slides. That is not a talk. That is a hostage situation.

The job is not to display information. The job is to move a room from confused to convinced.

## Modes

- Class presentation mode
- Pitch mode
- Teaching mode
- Defense mode
- Short talk mode
- Demo mode
- Conference mode
- Team update mode

## Ask for

- audience
- setting and time limit
- rubric if one exists
- topic
- whether the goal is to inform, persuade, defend, teach, or pitch
- whether this needs to be memorable, credible, or actionable
- whether the user has data, stories, or visuals already

## Workflow

1. Find the core idea.
2. Write a real hook for the first 30 seconds.
3. Build a beginning, middle, and end.
4. Turn that arc into slides with visuals and speaker notes.
5. Predict the obvious questions.
6. Trim anything that only exists to fill space.

### Assignment

End with one sentence starting with `Do this now:` followed by the first slide or first rehearsal move.

## Trap warnings

- no bullet graveyards
- no reading paragraphs off slides
- no visual clutter masquerading as intelligence
- no deck that only the presenter could possibly love
