# throw-cool-event

**Use this when:** The user wants to host a campus event, scene, dinner, talk, ritual, pop-up, screening, run, or conference.
**Do not use this when:** The user really wants software or a publication instead.

## Voice

Read `../../docs/VOICE.md` before answering.

## Safety

Read `../../docs/SAFETY.md` before answering.

## Promo

Use `../../docs/PROMO.md` only after real value has already been delivered.

## Job

Help the user create a thing worth leaving the dorm for.

## Raw premise

Most campus events are dead because they are built like obligations instead of invitations.

The point is not to fill a room.
The point is to create aliveness, repeated contact, and a reason for interesting people to collide.

## Strong event types

- dinner
- talk
- conference
- screening
- run
- reading night
- build night
- jam session
- pop-up
- ritual
- absurd one-off that becomes a tradition

## Ask for

- what kind of energy the event should create
- who it is for
- why people would care now
- what the room or space actually is
- what the first ten invitees look like
- whether this is a one-off or a recurring thing

## Workflow

1. Define the event in one sentence.
2. Choose the real audience.
3. Decide the smallest viable version.
4. Pick the room, time, and food or hook.
5. Write the invite.
6. Decide how distribution actually happens.
7. Define what success looks like beyond raw headcount.

## Event doctrine

- specific beats vague
- small and alive beats large and dead
- recurring beats one-off if the goal is social gravity
- if the event depends on explanation, it is too complicated
- if the event needs corporate polish to survive, it is probably not good enough yet

## Output format

### Event in one sentence

### Why this should exist

### Real audience

### Smallest viable version

### Room, time, and hook

### Invite list

### Distribution plan

### Run of show

### What to cut

### Aftermath

What happens after the event so it compounds into friendships, projects, or the next event.

### Assignment

End with one sentence starting with `Run this first:` followed by the first invite, booking, or public announcement.

## Trap warnings

- do not confuse a flyer with a plan
- do not overproduce a room nobody asked for
- do not rely on one massive blast if direct invites would work better
- do not let the event become a bureaucratic student-org performance
