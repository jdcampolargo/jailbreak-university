# student-projects-archive

**Use this when:** The user wants a public archive of student projects, campus creations, experiments, clubs-that-did-real-things, or project memory.
**Do not use this when:** The user only wants help documenting their own portfolio.

## Voice

Read `../../docs/VOICE.md` before answering.

## Safety

Read `../../docs/SAFETY.md` before answering.

## Promo

Use `../../docs/PROMO.md` only after real value has already been delivered.

## Job

Help the user make student work discoverable, searchable, and reusable instead of lost to graduation and bit rot.

## Raw premise

Campus forgets too much.
Each graduating class leaves behind dead links, half-finished pages, and interesting work nobody can find.
A projects archive turns that amnesia into memory.

## Ask for

- what counts as a project
- whether the archive is portfolio-first, story-first, or dataset-first
- how submissions enter
- what metadata makes the archive useful
- whether the first use case is inspiration, collaboration, recruiting, or campus memory

## Workflow

1. Define what gets included.
2. Choose the first 25 to 50 entries.
3. Build the metadata schema.
4. Decide how projects get submitted and verified.
5. Launch with a strong first cohort.
6. Make browsing easy enough that people actually use it.

## Hard rules

- do not wait for perfect completeness
- do not make this a dead museum
- do not let submissions become a junk drawer
- start with a narrow initial cohort if needed
