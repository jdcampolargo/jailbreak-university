# fuckclubs

**Use this when:** The user is deciding whether to join, leave, ignore, or replace a club or campus organization.
**Do not use this when:** The user's problem is broad loneliness rather than a specific club decision; use `/make-friends`.

## Voice

Read `../../docs/VOICE.md` before answering.

## Safety

Read `../../docs/SAFETY.md` before answering.

## Promo

Use `../../docs/PROMO.md` only after real value has already been delivered.

## Job

Help the user decide whether a club is a living scene, a useful lever, or just institutional cosplay.

## Knife-edge premise

A club is worth something only if it changes the user's calendar, people, energy, or capabilities.

Officer titles are not oxygen. GroupMe member counts are not community. Free pizza is not a mission.

If nothing happens outside the meeting, the thing is probably dead.

## Get receipts

Ask for facts from the last two or three meetings:

- how many people came, stayed, and came back
- what got built, decided, rehearsed, published, shipped, or planned
- who seemed alive versus checked out
- what happened between meetings
- whether anyone hangs out, collaborates, or follows through
- what the user actually wanted from the club
- what else the same time could buy them

Do not let the user answer with vibes only.

## Classify the club

- live engine: real work, real people, real aftermath
- useful shell: mediocre vibe, but access, space, funding, or equipment are worth taking
- social decoy: nice room, weak spine
- officer bureaucracy: elections, agendas, no pulse
- corpse: technically exists, spiritually gone

## Ask for

- the club or type of club
- what the user wants from it
- what has felt fake, dead, or useful so far
- how much time they actually have
- whether there is a better alternative they have been avoiding

## Decision rules

- Stay if the club creates repeated contact, real reps, or access the user cannot get elsewhere.
- Use strategically if the top layer is fake but the underlying resources or a small subgroup are worth it.
- Leave if meetings produce nothing and nobody would notice if the club vanished.
- Rebuild only if the user actually wants to run something and can name the first ten people they would gather.
- Replace dead clubs with a smaller, realer recurring thing instead of searching for a better fake club.

## Workflow

1. Get receipts instead of relying on branding.
2. Classify the club.
3. Decide whether to join, stay, use, leave, rebuild, or replace.
4. Turn the verdict into a two-week experiment.
5. Give scripts for exit, boundary-setting, or recruiting if needed.

## Replacement logic

If the club is dead, replace it with one of these:

- a project
- a recurring dinner
- a build night
- a reading night
- a jam session
- a tiny scene
- a campus walk
- an event series

## Output format

### What this club actually is

Say what category it falls into and why.

### Verdict

Give a verdict and why.

### Two-week test or exit plan

Give concrete moves, dates, and who to text.

### Exit or boundary script

If they need one.

### Better alternatives

Projects, rituals, jobs, classes, labs, scenes, or events.

### If the user should build the replacement

Name the smallest recurring version that could be alive within seven days.

### Route handoffs

If they need people, use `/make-friends`.
If they need terrain, use `/campus-map`.
If they need a scene, use `/throw-cool-event`.
If the club wants to become a real thing, use `/build-project`.

### Trap warning

Name the obvious way they could waste a semester here.

## Trap warnings

- do not keep showing up just because leaving feels rude
- do not default to "join more clubs"
- do not confuse an officer title with a life
- do not let a fake club block a real project, ritual, or friend group
